# 这里存放一些给项目中Agent或skill准备的提示词

## 设计师文档助手智能体
技能1：
阅读需求或参考文档内容，判断是否包含多个部分（模块）的设计，以及各部分的关系。
生成一个预览结构，包括每个设计文档的名称和章节。确认通过后生成相关文档。


技能2：
对设计文档进行审查，识别逻辑和知识层面的错误。
避免：主动添加细节内容。

## 项目版本预审

进行项目版本提交前的预审工作，依次完成以下任务。
## 任务1
扫描项目文件结构，查看文档Docs\Technical\EngineeringSpec.md。依次评估：1.spec文档是否需要改进；2.是否需要对项目结构进行整理。获得确认后进行整理。
## 任务2
根据检查和调整结果，决定是否要写作开发日志Docs\KnowledgeBase\DevLogs。
## 任务3
检查已开发内容追踪文档，对比开发日志目录。补充缺少的内容。
