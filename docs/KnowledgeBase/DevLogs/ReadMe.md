# 开发纪要(Dev Logs)写作规则
- **触发命令**：收到类似“写作开发日志”、“生成开发纪要”等请求时。
- **存放位置及参考模板**：`Docs/KnowledgeBase/DevLogs/_Template.md`
- **文件命名规则**：`YYYYMMDD_当日编号_任务和功能名称.md` (例如: `20251215_01_初始环境搭建.md`)，任务和功能名称默认使用中文。当日编号从01开始计数，如果存在相同日期的开发纪要，当日编号加1。
- **后续工作**：完成开发纪要写作后，更新<已开发内容追踪>文件。


## 已开发内容追踪 (Changelog)
- **存放文件**：`Docs/KnowledgeBase/ImplementedFeatures.md`
- **更新规则**：
  - 每次创建或更新Devlog后更新此文件。
  - 一定一定一定要保持极致的简洁，此文件的规模会随日志逐步增大。
